O programa utiliza as atribuições semânticas da ferramenta Bison para formar a árvore de derivação, conforme são feitas operações de redução no programa de entrada.
Em cada produção da gramática, são utilizadas as pseudo-variáveis ($0, $1) para obter o valor de cada terminal ou não terminal, e assim formar os nodos da árvore.
No arquivo de definição do parser (tiger.y), foi definida uma union contendo os tipos de dados que podem ser retornados pelo parser: int, char e tree. Cada token da linguagem
foi relacionado a um tipo de dado.

Para a criação da árvore, foi definida uma estrutura de dados no arquivo st.h, chamada tree. Um nodo de tree pode ser do tipo ID, Number, Operador (2 filhos),
Operador Múltiplo (mais de 2 filhos), Word (palavra reservada) e Terminal (tipos que não se encaixaram em nenhum dos outros anteriores). Na estrutura da árvore,
há uma union para garantir que cada nodo seja apenas de um tipo.
Em seguida, há a definição das funções de criação de nodos. Cada tipo de nodo possui uma função própria.

Erros conhecidos:

Como não encontrei uma especificação oficial para a linguagem Tiger--, algumas regras da gramática podem estar diferentes do que o esperado, gerando conflitos.
